## The code-block readout extension (one-line summary)

When `compose_graph_readout` walks a grounded conclusion and one of its
cited supports contains code, the support's full text is appended as an
indented fenced block right under the conclusion. Deduped per `sup_id`
so the same `_apply` node prints at most once across the whole answer.

Detection is a single regex (`_CODE_DETECT_RE` in `answerer_v2.py`)
matching `#include`, `std::`, `int main(`, `vector<`, `template<`,
`long long <name>(`. Backward compatible: any graph without code-bearing
support text (physics1, cs4, all the novelty-eval graphs) is unaffected
because the regex never matches.

Rationale lives in PROGRESS.md under the 2026-05-18 entry — the short
version is "an LLM paraphrasing C++ will hallucinate variable names; the
graph already had the right code, so make the readout surface it
verbatim instead of relying on the model's paraphrase."
